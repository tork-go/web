# Forms and uploads

A field tagged `form` reads a form-encoded request body:

```go
type LoginInput struct {
	Username string `form:"username"`
	Password string `form:"password"`
	Remember bool   `form:"remember"`
}

func login(ctx context.Context, in LoginInput) (Session, error)
```

```sh
curl -X POST localhost:8080/login \
  -d 'username=ada&password=secret&remember=on'
```

Both `application/x-www-form-urlencoded` and `multipart/form-data` bind the same
way, so a handler does not care which the client sent. `remember=on` binds to a
`bool` because that is what a checkbox sends.

Names derive from the field when the tag is empty, and the types are the same
ones every source binds — see
[Query parameters](/docs/2.fundamentals/4.query-parameters#types).

## Uploads

A field of type `*multipart.FileHeader` is an uploaded file:

```go
type AvatarInput struct {
	UserID string                `path:"user_id"`
	Note   string                `form:"note"`
	Avatar *multipart.FileHeader `form:"avatar"`
}

func setAvatar(ctx context.Context, in AvatarInput) (Result, error) {
	file, err := in.Avatar.Open()
	if err != nil {
		return Result{}, err
	}
	defer file.Close()

	return store.Save(ctx, in.UserID, in.Avatar.Filename, file)
}
```

`[]*multipart.FileHeader` takes every file sent under one name:

```go
type UploadInput struct {
	Files []*multipart.FileHeader `form:"files"`
}
```

A file field nothing was uploaded to is left nil rather than treated as an
error, so an optional attachment needs no special handling.

Tagging a file field with anything but `form` is a build error — no other source
can carry one.

## Size

The same `MaxBodyBytes` limit that applies to
[JSON bodies](/docs/2.fundamentals/6.request-bodies#size) applies here, and it
is the one to raise for an endpoint that takes uploads:

```go
app := tork.New(tork.MaxBodyBytes(20 << 20))
```

Up to 10 MiB of a multipart request is held in memory; the rest goes to
temporary files, which `net/http` cleans up when the request ends. That is a
buffering threshold, not a limit on the request — `MaxBodyBytes` is the limit.

## Content types

A form endpoint requires a form content type. A request that sends JSON, or none
at all, answers **415**:

```json
{
  "code": "UNSUPPORTED_MEDIA_TYPE",
  "message": "This endpoint reads a form, not \"application/json\".",
  "status": 415,
  "timestamp": "2026-07-23T16:37:23Z",
  "path": "/login"
}
```

A body that claims to be multipart but is not readable as one answers 400.

## Forms and bodies do not mix

A form and a JSON body both consume the request, which can only be read once, so
declaring both in one handler is a build error:

```
auth/handler.go:11: a form field cannot be read when items.UpdateItemBody is
    already the request body; a request body is read once
```
